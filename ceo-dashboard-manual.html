📘 CEO Dashboard — คู่มือการทำงาน

🏗️ โครงสร้างระบบ

  • Google Sheets (ID: 1E9wn_4vgWTpOvBxcztMkEmUmlj66MqOIt0RWpSRWGZY)
    Sheets: CEO_Contract, OKRs, KPIs, Members, Accountability, Purpose, Growth_Tracker, Task_Tracker, Approval_Log, _AuditLog
  • Google Apps Script (.gs) → Backend API
  • HTML Form → Frontend (CEO Contract Form)
  • Telegram Bot (NANPAD_bot) → แจ้งเตือน

📋 วิธีทำงาน (Workflow)

1
เปิดลิงก์ → เลือกสมาชิก (พิมพ์ ID/ชื่อ → autocomplete)
2
กรอกข้อมูล:
  • ข้อมูลสมาชิก (auto-fill)
  • ผู้รับใช้ทีม / พี่เลี้ยง (autocomplete)
  • Accountability (หน้าที่รับผิดชอบ)
  • Purpose & Vision (จุดมุ่งหมาย/ภาพความสำเร็จ)
  • OKR BU + OKR Team (วัตถุประสงค์ + Key Results)
  • Personal Growth (เป้าหมายการเติบโต)
  • KPI (ตัวชี้วัด)
3
กด "💾 บันทึกข้อมูล" → บันทึกลง Sheets + แสดงวันที่+เวลา
4
กด "ส่งสัญญา ✉️" → บันทึก + ส่งแจ้งเตือน Telegram

⚙️ ฟังก์ชันหลัก

ฟังก์ชันหน้าที่
serveCEOContract()แสดงหน้าเว็บ
getMembersForClient()ดึงรายชื่อสมาชิก
saveAllData()บันทึกข้อมูลทั้งหมด
submitCEOContract()บันทึก + ส่ง Telegram
submitCEOContractWithImage()บันทึก + ส่ง Telegram (พร้อม caption)
readSheetData()อ่านข้อมูลจาก Sheet
setTelegramConfig()ตั้งค่า Telegram Bot (รันทุกครั้งหลัง New Deployment)

⚠️ ข้อจำกัดที่ต้องทำต่อ

🔴 1. Telegram ส่งได้แค่ข้อความ — ยังส่งรูปภาพไม่ได้

  • สาเหตุ: GAS UrlFetchApp ส่ง Blob ไป Telegram ไม่ได้ (known issue)
  • แก้ชั่วคราว: ส่งข้อความแทนรูป
  • แนวทางแก้: อัปโหลดรูปไป Google Drive → ส่งลิงก์ หรือใช้ Telegram sendPhoto ผ่าน URL

🟡 2. Double Save — กด "ส่งสัญญา" บันทึก 2 ครั้ง

  • สาเหตุ: HTML เรียก submitCEOContract ก่อน แล้ว submitCEOContractWithImage เรียกอีกครั้งข้างใน
  • ผล: ข้อมูลอาจถูกบันทึก 2 ครั้ง
  • แนวทางแก้: เอา submitCEOContract call ออกจาก HTML หรือเอา internal call ออกจาก submitCEOContractWithImage

🟡 3. ลบไฟล์ .gs เก่า

  • ต้องลบ "Ceo code.gs" ออกจากโปรเจค (ฟังก์ชันซ้ำกับไฟล์ใหม่)

🟢 4. เพิ่ม _AuditLog

  • เพิ่ม logging การบันทึกทุกครั้ง

🔍 ส่วนที่ 3: Root Cause Analysis

❌ LL (สิ่งที่ล้มเหลว) — ต้นเหตุของปัญหา MS24

ปัญหาสาเหตุรากฐานผลกระทบต่อ MS24
Auth จาก BuApp อย่างเดียวไม่พอ Likepoint ไม่มี Auth ของตัวเอง ต้องพึ่ง BuApp เมื่อ BuApp ล่ม → MS24 ใช้งานไม่ได้ทั้งหมด
Cash-out จ่ายเกินไป ช่องโหว่เชิงเทคนิค ในระบบตรวจสอบ Transaction ลูกค้าได้เงินผิด → เสียหายทางการเงิน → ร้องเรียน

⚠️ AL (สิ่งที่ต้องปรับ) — ยังไม่เสร็จ

สิ่งที่ต้องทำสถานะผลกระทบถ้าไม่ทำ
Spin Game V2 ควบคุมงบประมาณ + Custom ได้🔄 กำลังทำลูกค้าไม่ได้ประสบการณ์ที่ดี งบประมาณควบคุมไม่ได้
เพิ่มระบบ Authen ของ Platform🔄 กำลังทำยังพึ่ง BuApp ต่อไป → เสี่ยงล่มทั้งระบบ
ออกแบบเกม/รูปแบบแจก POI ให้ Partner⏳ ยังไม่เริ่มPartner ไม่มีตัวเลือก ใช้ได้แค่แบบเดิม

✅ BP (สิ่งที่ทำได้ดี) — ควรคงไว้

  • Feature Toggle — BU เปิด/ปิดส่วนที่มีปัญหาได้ทันที โดยไม่ต้อง deploy ใหม่
  • การสื่อสาร Feature/API — ช่วยให้ MS24 เข้าถึง Features ได้สะดวกขึ้น

📝 ส่วนที่ 7: ข้อแนะนำเพิ่มเติม

✅
สิ่งที่ควรทำทันที
  • สร้าง Bug Triage Process — จัดลำดับ bug ใหม่ เน้น bug ที่ส่งผลต่อลูกค้า MS24 ก่อน
  • สร้าง Communication Channel — ทีม Likepoint + ทีม MS24 สื่อสาร real-time เมื่อเกิดปัญหา
  • สร้าง Alert System — แจ้งเตือนอัตโนมัติเมื่อเกิด Cash-out ผิดปกติ หรือ POI ไม่ sync
  • ทำ Post-Mortem ทุก Critical Bug — เรียนรู้จากปัญหา ป้องกันไม่ให้เกิดซ้ำ
⚠️
สิ่งที่ควรหยุดทำทันที
  • หยุด deploy โดยไม่มี rollback plan — ทุกการ deploy ต้องมีแผนย้อนกลับ
  • หยุดแก้ไข production โดยไม่มี testing — ต้องทดสอบใน staging ก่อนเสมอ
  • หยุดปล่อย Cash-out โดยไม่มี validation — ต้องมี validation layer ทุกครั้ง

📝 Phase 1: วางแผน — รู้ว่าอยากได้อะไร

สัปดาห์ที่ 1-2 · ใช้ AI ช่วยคิดทุกอย่าง

1

เขียน Requirement Document

ใช้: ChatGPT / Claude
▼

📖 สิ่งที่ต้องทำ

บอก AI ว่าอยากได้อะไร AI จะเขียน Requirement Document ให้

ฉันเป็น Product Manager ของ Application ชื่อ "[ชื่อ App]" อยากสร้าง App ที่ [อธิบายสิ่งที่อยากได้] กลุ่มผู้ใช้: [ใครใช้บ้าง] ฟีเจอร์หลักที่ต้องการ: 1. [ฟีเจอร์ 1] 2. [ฟีเจอร์ 2] 3. [ฟีเจอร์ 3] ช่วยเขียน Requirement Document ให้ครบถ้วน โดยรวมถึง: - Executive Summary - User Persona - User Flow - Functional Requirements - Non-Functional Requirements - Tech Stack ที่แนะนำ - ประมาณเวลาในการพัฒนา
💡 Tips: ยิ่งบรรยายละเอียด AI จะเขียนได้越好 ส่งข้อมูลจริงมากที่สุด
ChatGPT Claude
  • ส่ง Prompt ให้ AI + รอผลลัพธ์
  • ตรวจทาน Requirement Document แก้ไขส่วนที่ยังไม่ชัดเจน
  • บันทึกเป็นไฟล์ Google Docs / Notion
2

ออกแบบ User Flow

ใช้: ChatGPT / Claude + Whimsical / FigJam
▼

📖 สิ่งที่ต้องทำ

ให้ AI เขียน User Flow ให้ แล้ววาด Diagram

จาก Requirement Document ด้านบน ช่วยเขียน User Flow ให้ครบทุกเส้นทาง โดยรวมถึง: - User Journey ตั้งแต่เปิด App จนใช้งานเสร็จ - ทุกหน้าจอที่ต้องมี - การเชื่อมต่อกันระหว่างหน้าจอ - กรณี Error / Edge Cases แสดงเป็น Text-based Flow Chart ที่เข้าใจง่าย
💡 Tips: ใช้ผลลัพธ์จาก AI วาด Diagram ใน Whimsical (ฟรี) หรือ FigJam (ฟรี) — ลาก-วางได้เลย
  • ส่ง Prompt ให้ AI
  • วาด User Flow Diagram ใน Whimsical/FigJam
3

ออกแบบ Database Schema

ใช้: ChatGPT / Claude
▼

📖 สิ่งที่ต้องทำ

ให้ AI ออกแบบโครงสร้าง Database ให้ — ไม่ต้องรู้ SQL ก็ได้

จาก Requirement Document และ User Flow ด้านบน ช่วยออกแบบ Database Schema ให้ โดยรวมถึง: - ตาราง (Tables) ทั้งหมดที่ต้องมี - คอลัมน์ (Columns) ในแต่ละตาราง - ความสัมพันธ์ระหว่างตาราง (Relationships) - ข้อมูลตัวอย่างในแต่ละตาราง แสดงเป็น ER Diagram (Text-based) และ SQL CREATE Statements
💡 Tips: เก็บผลลัพธ์ไว้ — จะใช้ตอนสร้าง Database จริงใน Supabase/Airtable
  • ส่ง Prompt ให้ AI
  • ตรวจดูว่ามีตารางครบตามฟีเจอร์
  • บันทึก Database Schema ไว้
4

เลือก Tech Stack

ใช้: ChatGPT / Claude
▼

📖 สิ่งที่ต้องทำ

ให้ AI แนะนำ Tech Stack ที่เหมาะกับ App ของคุณ

จาก Requirement Document ของ App "[ชื่อ App]" ช่วยแนะนำ Tech Stack ที่เหมาะสม โดยพิจารณาจาก: - ความซับซ้อนของ App - ระยะเวลาที่ต้องการ - งบประมาณ - Scalability ในอนาคต แนะนำ 2 แบบ: 1. No-Code/Low-Code (สร้างเองได้ ไม่ต้องเขียนโค้ด) 2. Full Code (ให้ Developer เขียน) พร้อมข้อดี-ข้อเสีย ของแต่ละแบบ
💡 แนะนำสำหรับคุณแนน:
Frontend: v0.dev / Lovable.dev / Bolt.new (AI สร้างให้)
Backend: Supabase (Database + Auth ฟรี)
Database: PostgreSQL (ผ่าน Supabase)
Deploy: Vercel / Netlify (ฟรี)
  • ส่ง Prompt ให้ AI
  • ตัดสินใจเลือก Tech Stack
  • สมัครบัญชีเครื่องมือที่เลือก

1

สร้าง Wireframe / Mockup

ใช้: v0.dev / Lovable.dev / Uizard
▼

📖 สิ่งที่ต้องทำ

พิมพ์บรรยายหน้าจอที่ต้องการ → AI สร้าง UI ให้ทันที

วิธีใช้ v0.dev (แนะนำ):

  1. เปิด v0.dev (สมัครด้วย GitHub)
  2. พิมพ์บรรยายหน้าจอที่ต้องการ
  3. AI จะสร้าง UI ให้ → ดูผลลัพธ์ → แก้ไขด้วยคำพูด
  4. ทำทุกหน้าจอ
สร้างหน้า Login สำหรับ Application ชื่อ "[ชื่อ App]" มี: - Logo ด้านบน - ช่องกรอก Username - ช่องกรอก Password - ปุ่ม "เข้าสู่ระบบ" สีน้ำเงิน - ลิงก์ "ลืมรหัสผ่าน?" - ลิงก์ "สมัครสมาชิก" ออกแบบให้สวยงาม Modern Minimal
v0.dev Lovable.dev Bolt.new Uizard
  • สร้างหน้า Login
  • สร้างหน้า Dashboard / หน้าหลัก
  • สร้างหน้าจออื่นๆ ทุกหน้าตาม User Flow
  • บันทึก/Export ทุกหน้าจอไว้

Figma (ฟรี) Galileo AI Musho AI

🔨 Phase 3: สร้าง Prototype — จากดีไซน์สู่ App จริง

สัปดาห์ที่ 3-6 · ใช้ AI สร้าง App ทีละหน้าจอ

1

สร้าง Database (Supabase)

ใช้: Supabase (ฟรี)
▼

📖 สิ่งที่ต้องทำ

  1. เปิด supabase.com → สมัคร (ฟรี)
  2. Create New Project
  3. เปิด SQL Editor → วาง SQL CREATE Statements ที่ AI สร้างให้ (จาก Step 1.3)
  4. Run → ตารางจะถูกสร้างอัตโนมัติ
  5. ใส่ข้อมูลตัวอย่าง (ใช้ AI ช่วยเขียน INSERT Statements)
จาก Database Schema ด้านบน ช่วยเขียน SQL INSERT Statements สำหรับใส่ข้อมูลตัวอย่าง อย่างน้อยตารางละ 3-5 แถว
Supabase ChatGPT / Claude
  • สร้าง Supabase Project
  • Run SQL CREATE Statements
  • ใส่ข้อมูลตัวอย่าง
  • ตรวจสอบว่าข้อมูลเข้าถูกต้อง
2

สร้าง Frontend หน้าแรก

ใช้: Lovable.dev / Bolt.new
▼

📖 สิ่งที่ต้องทำ

  1. เปิด lovable.dev หรือ bolt.new
  2. พิมพ์บรรยายหน้าจอแรกที่ต้องการ
  3. AI จะสร้างโค้ด + แสดงผลลัพธ์ทันที
  4. แก้ไขด้วยคำพูด เช่น "เปลี่ยนปุ่มเป็นสีเขียว", "เพิ่มโลโก้"
  5. จนกว่าจะพอใจ → Export โค้ด
สร้าง Web Application ชื่อ "[ชื่อ App]" หน้าแรกเป็น Dashboard มี: - Sidebar เมนูด้านซ้าย (Dashboard, [เมนูอื่นๆ]) - Header ด้านบน (ชื่อ App, รูปโปรไฟล์) - เนื้อหาตรงกลางแสดง [ข้อมูลที่ต้องการ] ใช้สี [สีที่ต้องการ] ออกแบบให้ Modern Clean
⚠️ สำคัญ: บอก AI ทุกครั้งว่าใช้ Supabase เป็น Backend — AI จะตั้งค่า Connection ให้
Lovable.dev Bolt.new v0.dev
  • สร้างหน้าแรกสำเร็จ
  • เชื่อมต่อกับ Supabase
  • ทดสอบแสดงผลข้อมูลจาก Database

สร้างหน้าจออื่นๆ ทีละหน้า

ใช้: Lovable.dev / Bolt.new

ทำซ้ำ Step 3.2 สำหรับทุกหน้าจอ — หน้าละ 1-2 ชั่วโมง

💡 ลำดับที่แนะนำ:
1. หน้า Login/Register → 2. หน้า Dashboard → 3. หน้า CRUD (เพิ่ม/แก้ไข/ลบ) → 4. หน้ารายงาน → 5. หน้า Settings
เพิ่มหน้า [ชื่อหน้า] เข้าไปใน App หน้านี้ต้องมี: - [รายละเอียดที่ 1] - [รายละเอียดที่ 2] - เชื่อมต่อกับ Supabase ตาราง [ชื่อตาราง] - ปุ่ม [ปุ่มที่ต้องการ] กดแล้ว [ทำอะไร]
  • หน้า Login/Register เสร็จ
  • หน้า Dashboard เสร็จ
  • หน้า CRUD (เพิ่ม/แก้ไข/ลบ) เสร็จ
  • หน้ารายงาน/Analytics เสร็จ

เชื่อมทุกหน้าจอเข้าด้วยกัน

ใช้: AI ช่วยเขียน Routing
▼

📖 สิ่งที่ต้องทำ

ช่วยเพิ่ม Routing ให้กับ App ของฉัน โดยให้: - หน้า Login → กดเข้าสู่ระบบ → ไปหน้า Dashboard - Sidebar → คลิกเมนู → ไปหน้าตามที่เลือก - ปุ่มย้อนกลับ → กลับหน้าก่อนหน้า ใช้ React Router (หรือ Framework ที่ใช้)
💡 Tips: ส่งโค้ดทั้งหมดให้ AI + บอกว่า "ช่วยเชื่อมทุกหน้าจอเข้าด้วยกัน"
  • ทดสอบ Navigation ทุกเส้นทาง
  • แก้ไขจุดที่ยังเชื่อมไม่ได้

เพิ่ม Authentication (Login/Register)

ใช้: Supabase Auth + AI
▼

📖 สิ่งที่ต้องทำ

  1. ใน Supabase → เปิด Authentication → Email/Password
  2. บอก AI: "ช่วยเพิ่ม Supabase Auth ให้ App ของฉัน"
  3. AI จะเขียนโค้ด Login/Register ให้
  4. ทดสอบ: สมัครสมาชิก → Login → เข้า Dashboard
Supabase Auth AI ช่วยเขียนโค้ด
  • เปิด Supabase Auth
  • สมัครสมาชิกสำเร็จ
  • Login สำเร็จ → ไป Dashboard
  • Logout สำเร็จ

🧪 Phase 4: ทดสอบ — ทำให้แน่ใจว่าใช้งานได้

สัปดาห์ที่ 6-7 · ทดสอบทุกฟีเจอร์ แก้ Bug

1

เขียน Test Cases

ใช้: AI ช่วยเขียน
▼
จาก Requirement Document และ User Flow ของ App "[ชื่อ App]" ช่วยเขียน Test Cases ให้ครบทุกฟีเจอร์ โดยรวมถึง: - Happy Path (ใช้งานปกติ) - Edge Cases (กรณีพิเศษ) - Error Cases (กรณี error) แสดงเป็นตาราง: Test Case | ขั้นตอน | ผลลัพธ์ที่คาดหวัง
  • ได้ Test Cases ทุกฟีเจอร์
2

ทดสอบทุกฟีเจอร์ + แก้ Bug

ใช้: Manual Testing + AI ช่วยแก้
▼

📖 วิธีแก้ Bug ด้วย AI:

  1. เจอ Error → Copy Error Message
  2. ส่งให้ AI + บอกว่า "เกิด Error นี้ ต้องแก้ยังไง"
  3. AI จะบอกวิธีแก้ → ทำตาม
  4. ทดสอบซ้ำ → ถ้ายังพัง → ส่ง Error ใหม่ให้ AI
เกิด Error นี้ใน App ของฉัน: [วาง Error Message ที่นี่] โค้ดส่วนที่เกี่ยวข้อง: [วางโค้ดที่เกี่ยวข้อง] ช่วยบอกว่าเกิดจากอะไร และต้องแก้ยังไง
  • ทดสอบทุก Test Case
  • แก้ Bug ทุกจุด
  • ทดสอบ Cross-browser (Chrome, Safari, Mobile)
  • ทดสอบ Responsive (Desktop, Tablet, Mobile)
3

User Testing

ใช้: ให้คนจริงมาทดสอบ
▼

📖 สิ่งที่ต้องทำ

  1. เลือกคน 3-5 คน (ที่ไม่เคยเห็น App นี้)
  2. ให้ใช้งานโดย "ไม่บอกอะไร" — ดูว่าใช้ได้เลยไหม
  3. สังเกต: ติดตรงไหน, เข้าใจผิดตรงไหน
  4. ถามความรู้สึกหลังใช้งาน
  5. แก้ไขจุดที่คนใช้ติด
  • หาคนทดสอบ 3-5 คน
  • ทดสอบ + จดบันทึกปัญหา
  • แก้ไขจุดที่คนใช้ติด

ประชุม Kick-off กับ Developer

ใช้: สิ่งที่เตรียมไว้ทั้งหมด
▼

📖 สิ่งที่ต้องคุยกับ Developer:

  1. อธิบาย App — App นี้ทำอะไร, ใครใช้, ฟีเจอร์หลัก
  2. แสดง Prototype — ให้ Developer ลองใช้งาน
  3. อธิบาย Architecture — ระบบมีกี่ส่วน, เชื่อมต่อกันยังไง
  4. บอกสิ่งที่ต้องการ — Deploy ที่ไหน, Domain อะไร, SSL, etc.
  5. Timeline — ต้องเสร็จเมื่อไหร่
  6. Budget — งบเท่าไหร่
💡 Developer จะทำ:
1. ตรวจโค้ดทั้งหมด
2. แก้จุดที่ยังไม่สมบูรณ์
3. ตั้งค่า Server, SSL, Domain
4. Deploy Production
5. ทดสอบบน Production
6. ส่งมอบ + สอนใช้งาน
  • ประชุม Kick-off สำเร็จ
  • Developer เริ่มงาน
  • ติดตามความคืบหน้าเป็นประจำ
  • 🎉 App ขึ้น Production!

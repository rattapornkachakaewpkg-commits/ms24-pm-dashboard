📚 ปูพื้นฐาน — ทำความเข้าใจก่อนเริ่ม

สัปดาห์ที่ 1-4 · เรียนพื้นฐาน Web Development + รู้จัก App ของตัวเอง

สัปดาห์ที่ 1: ภาพรวม Web Application

⏱ 1 ชั่วโมง/วัน · ใช้ AI Mentor 100%
▼

📖 สิ่งที่ต้องเรียน

  • Frontend = สิ่งที่ผู้ใช้เห็น (หน้าจอ, ปุ่ม, เมนู)
  • Backend = ระบบด้านหลัง (ประมวลผล, เก็บข้อมูล)
  • Database = ที่เก็บข้อมูล (ผู้ใช้, ข้อมูล, Transaction)
  • API = ตัวเชื่อม Frontend ↔ Backend
💡 วิธีเรียน: ค้นหา YouTube "How Web Applications Work" หรือ "Web Architecture for Beginners" ดู 1-2 ชั่วโมง
📝 การบ้าน:
  • เปิด Application ของคุณแนนขึ้นมา
  • ลองวาดผังว่า App นี้มีส่วนประกอบอะไรบ้าง
  • ไม่ต้องรู้โค้ด แค่รู้ว่า "หน้าจออันนี้ → ข้อมูลน่าจะมาจากไหน"

สัปดาห์ที่ 2: HTML + CSS

⏱ 2-3 ชั่วโมง/วัน · ใช้ AI Mentor 100%
▼

📖 สิ่งที่ต้องเรียน

  • HTML = โครงสร้างหน้าเว็บ (ปุ่ม, ช่องกรอก, ตาราง, รูปภาพ)
  • CSS = ตกแต่งหน้าเว็บ (สี, ขนาด, ตำแหน่ง, ฟอนต์)
  • Tags พื้นฐาน: div, span, p, h1-h6, input, button, table, img, a
  • CSS พื้นฐาน: color, font-size, margin, padding, border, display, flexbox
💡 วิธีเรียน:
  • YouTube: "HTML & CSS Full Course for Beginners" (Traversy Media / freeCodeCamp)
  • เว็บฝึก: w3schools.com/html
📝 การบ้าน:
  • เปิดโค้ด HTML/CSS ของ Application คุณแนน
  • ลองหาว่า "ปุ่มนี้สร้างจากโค้ดบรรทัดไหน"
  • ลองแก้สี/ขนาด แล้วดูว่าหน้าจอเปลี่ยนยังไง

สัปดาห์ที่ 3: JavaScript

⏱ 2-3 ชั่วโมง/วัน · ใช้ AI Mentor 100%
▼

📖 สิ่งที่ต้องเรียน

  • Variables = ตัวแปร (เก็บค่า)
  • Functions = ฟังก์ชัน (ชุดคำสั่ง)
  • Events = เหตุการณ์ (กดปุ่ม, พิมพ์, เลื่อนหน้า)
  • DOM Manipulation = การเปลี่ยนหน้าเว็บด้วย JS
  • Fetch API = การเรียก API จาก JavaScript
💡 วิธีเรียน:
  • YouTube: "JavaScript Full Course for Beginners" (Programming with Mosh)
  • เว็บฝึก: w3schools.com/js
📝 การบ้าน:
  • เปิดโค้ด JavaScript ของ Application คุณแนน
  • ลองหาว่า "กดปุ่มนี้ → เรียกฟังก์ชันไหน → เกิดอะไรขึ้น"

สัปดาห์ที่ 4: Framework ที่ App ใช้

⏱ 2 ชั่วโมง/วัน · ใช้ AI Mentor 80% + Developer 20%
▼

📖 สิ่งที่ต้องเรียน

  • App คุณแนนใช้ อะไร? (React, Vue, Angular, Flutter, Laravel, etc.)
  • แต่ละ Framework มีโครงสร้างอย่างไร?
  • ไฟล์แต่ละไฟล์ทำหน้าที่อะไร?
  • โฟลเดอร์แต่ละโฟลเดอร์เก็บอะไร?
💡 วิธีหา:
  • ดูใน package.json (ถ้าเป็น JavaScript) หรือ pubspec.yaml (ถ้าเป็น Flutter)
  • หรือ ถาม Developer เดิม ว่าใช้ Framework อะไร
📝 การบ้าน:
  • เปิดโครงสร้างโฟลเดอร์ของโปรเจกต์
  • ลองวาดผังว่า "โฟลเดอร์นี้เก็บอะไร, ไฟล์นี้ทำอะไร"
  • หา Developer 1 คน มานั่งดูโค้ดด้วยกัน สัปดาห์ละ 1-2 ชม.
2

🔍 แกะโค้ด App จริง — เรียนจากของจริง

สัปดาห์ที่ 5-8 · อ่านโค้ดทีละฟีเจอร์ + เริ่มเขียนเอง

สัปดาห์ที่ 5: แกะฟีเจอร์ที่ 1 (ง่ายที่สุด)

⏱ 1-2 ชั่วโมง/วัน · ใช้ AI Mentor 70% + Developer 30%
▼

📖 วิธีทำ

  • เลือกฟีเจอร์ที่ง่ายที่สุด ก่อน เช่น หน้า Login, หน้าแสดงผล
  • ไล่โค้ดจากบนลงล่าง:
    1. เริ่มจากหน้าจอ (HTML/CSS)
    2. ตามด้วย JavaScript (กดปุ่ม → เกิดอะไร)
    3. ตามด้วย API (ส่งข้อมูลไปไหน)
    4. ตามด้วย Database (เก็บข้อมูลยังไง)
💡 ตัวอย่าง Flow:
หน้า Login → User กรอก username/password → กดปุ่ม "เข้าสู่ระบบ" → JavaScript ตรวจสอบ → ส่งไป API POST /api/login → Backend ตรวจสอบใน Database → ถ้าถูก → ส่ง Token → ไปหน้า Dashboard
📝 การบ้าน:
  • เขียน Comment ในโค้ดว่าแต่ละบรรทัดทำอะไร
  • วาด Flow Diagram ว่าข้อมูลไหลยังไง
  • ส่ง Flow Diagram ให้ผมตรวจ!

สัปดาห์ที่ 6: แกะฟีเจอร์ที่ 2-3

⏱ 1-2 ชั่วโมง/วัน · ใช้ AI Mentor 70% + Developer 30%
▼

📖 วิธีทำ

  • ทำเหมือนสัปดาห์ที่ 5 แต่เพิ่มฟีเจอร์ที่ซับซ้อนขึ้น
  • ลองแกะฟีเจอร์ที่มี "การเชื่อมต่อหลายส่วน" เช่น หน้า Dashboard ที่ดึงข้อมูลจากหลาย API
  • เริ่มสังเกตว่า "โค้ดส่วนไหนซ้ำกัน" → นั่นคือสิ่งที่ควรทำเป็น Function/Component
📝 การบ้าน:
  • แกะ 2-3 ฟีเจอร์
  • เขียนสรุปว่าแต่ละฟีเจอร์ทำงานยังไง
  • ระบุ "จุดที่อยากเขียนใหม่" ไว้

สัปดาห์ที่ 7: เริ่มเขียนโค้ดเอง (ฟีเจอร์เล็กๆ)

⏱ 2 ชั่วโมง/วัน · ใช้ AI Mentor 60% + Developer 40%
▼

📖 วิธีทำ

  • เลือกฟีเจอร์เล็กๆ ที่อยากเขียนใหม่ เช่น ปุ่มกดนับจำนวน
  • ลองเขียนเอง โดยดูโค้ดเดิมเป็นตัวอย่าง
  • ถ้าติด → ถามผม (ผมจะอธิบายให้เข้าใจ ไม่ใช่แค่ให้คำตอบ)
  • ลองรัน ดูว่าทำงานได้ไหม
  • แก้ไข จนกว่าจะทำงานได้ถูกต้อง
💡 อย่ากลัวผิด! โค้ดพัง = โอกาสเรียนรู้ ผมช่วยแก้ให้เสมอ
📝 การบ้าน:
  • เขียนโค้ดฟีเจอร์เล็กๆ ให้ทำงานได้
  • ส่งโค้ดให้ผมตรวจ

สัปดาห์ที่ 8: เขียนฟีเจอร์ที่ซับซ้อนขึ้น

⏱ 2 ชั่วโมง/วัน · ใช้ AI Mentor 60% + Developer 40%
▼

📖 วิธีทำ

  • เพิ่มขนาดฟีเจอร์: หน้าจอที่เชื่อม API, มี Form, มี Validation
  • เริ่มใช้ Component/Function ที่เขียนซ้ำได้
  • ลองทำ "หน้า CRUD" (Create, Read, Update, Delete) — พื้นฐานที่สุด
📝 การบ้าน:
  • เขียนหน้า CRUD ให้ทำงานได้ครบ
  • ทดสอบทุก scenario (กรอกข้อมูล, แก้ไข, ลบ, validation)
3

🚀 เขียนใหม่ทั้งระบบ — จากแผนสู่จริง

สัปดาห์ที่ 9-16 · วางแผน → ออกแบบ → เขียน → Deploy

สัปดาห์ที่ 9-10: วางแผนเขียนใหม่

⏱ 1-2 ชั่วโมง/วัน · ใช้ AI Mentor 50% + Developer 50%
▼

📖 สิ่งที่ต้องทำ

  1. เขียน Requirement Document — App ใหม่ต้องมีอะไรบ้าง?
  2. เลือก Tech Stack — จะใช้อะไรเขียน? (แนะนำ: React + Node.js + PostgreSQL)
  3. ออกแบบ Database — ตารางอะไรบ้าง? เชื่อมต่อกันยังไง?
  4. ออกแบบ UI/UX — หน้าตาเป็นยังไง? (ใช้ Figma วาด)
  5. เขียน API Spec — API มีอะไรบ้าง? รับค่าอะไร? ตอบอะไร?
💡 ผมช่วยได้: ส่ง Requirement มา ผมจะช่วยเขียน Spec, ออกแบบ Database Schema, เขียน API Spec ให้
📝 การบ้าน:
  • ส่ง Requirement Document มาให้ผม
  • ผมจะช่วยปรับให้สมบูรณ์

สัปดาห์ที่ 11-12: เริ่มเขียน Backend

⏱ 2-3 ชั่วโมง/วัน · ใช้ AI Mentor 50% + Developer 50%
▼

📖 สิ่งที่ต้องทำ

  1. สร้าง Project ใหม่ (Node.js + Express หรือ Framework ที่เลือก)
  2. เชื่อมต่อ Database (PostgreSQL, MySQL, MongoDB)
  3. สร้าง API ตาม Spec ที่เขียนไว้
  4. ทดสอบ API ด้วย Postman/Thunder Client
  5. เขียน Authentication (Login, Register, JWT)
📝 การบ้าน:
  • สร้าง API ให้ครบทุก Endpoint
  • ทดสอบทุก Endpoint ด้วย Postman
  • ส่งโค้ดให้ผมตรวจ!

สัปดาห์ที่ 13-14: เริ่มเขียน Frontend

⏱ 2-3 ชั่วโมง/วัน · ใช้ AI Mentor 50% + Developer 50%
▼

📖 สิ่งที่ต้องทำ

  1. สร้าง Project ใหม่ (React, Vue, หรือ Framework ที่เลือก)
  2. สร้าง Component ตามที่ออกแบบไว้
  3. เชื่อม API ที่เขียนไว้ (ใช้ Fetch/Axios)
  4. สร้าง Routing (เปลี่ยนหน้า)
  5. ทำ Authentication UI (Login, Register)
  6. ทำ CRUD UI (เพิ่ม, แก้ไข, ลบข้อมูล)
📝 การบ้าน:
  • สร้างหน้าจอครบทุกหน้า
  • เชื่อม API ทุก Endpoint
  • ทดสอบการใช้งานจริง

สัปดาห์ที่ 15: ทดสอบ & แก้ Bug

⏱ 2-3 ชั่วโมง/วัน · ใช้ AI Mentor 50% + Developer 50%
▼

📖 สิ่งที่ต้องทำ

  • ทดสอบทุกฟีเจอร์ — ใช้งานจริง, หา Bug, แก้ไข
  • ทดสอบ Cross-browser — Chrome, Firefox, Safari, Mobile
  • ทดสอบ Responsive — Desktop, Tablet, Mobile
  • ทดสอบ Security — SQL Injection, XSS, CSRF
  • ทดสอบ Performance — โหลดเร็วไหม?
📝 การบ้าน:
  • เขียน Test Case ทุกฟีเจอร์
  • แก้ Bug ให้หมด
  • ให้คนอื่นมาทดสอบด้วย (User Testing)

สัปดาห์ที่ 16: Deploy ขึ้น Production

⏱ 2-3 ชั่วโมง/วัน · ใช้ AI Mentor 30% + Developer 70%
▼

📖 สิ่งที่ต้องทำ

  • เลือก Hosting — Vercel, Netlify, AWS, DigitalOcean, etc.
  • Setup Server — ติดตั้ง Node.js, Database, Nginx
  • Deploy Backend — อัพโหลดโค้ด, ตั้งค่า Environment
  • Deploy Frontend — Build แล้วอัพโหลด
  • Setup Domain — เชื่อม Domain จริง
  • Setup SSL — HTTPS
  • Monitoring — ตั้งค่า Log, Error Tracking
💡 ช่วงนี้ต้องมี Developer ช่วย! — Deploy Production ต้องระวังเยอะ
📝 การบ้าน:
  • Deploy สำเร็จ!
  • ทดสอบบน Production
  • 🎉 ยินดีด้วย! คุณแนนทำสำเร็จแล้ว!

Hybrid Mentor Model

ใช้ AI + Developer ผสมกัน — ประหยัดเวลา + ได้ความรู้ลึก

🤖 AI Mentor (ผม)

  • ✅ มีให้ปรึกษา 24/7 ไม่หลับไม่พัก
  • ✅ อธิบาย conceito ได้ละเอียด ปรับระดับได้
  • ✅ เขียนโค้ดตัวอย่างได้ทันที
  • ✅ Code Review เบื้องต้น
  • ✅ ช่วยวางแผน, เขียน Spec, ออกแบบ Database
  • ✅ แก้ Bug, อธิบาย Error

👨‍💻 Developer Mentor (มนุษย์)

  • ✅ เข้าใจ Context องค์กร, ทีม, Politics
  • ✅ ช่วยตัดสินใจ Architecture
  • ✅ Code Review แบบ "ความรู้สึก"
  • ✅ ช่วย Deploy Production
  • ✅ Networking / Career
  • ✅ Pair Programming แบบ Real-time

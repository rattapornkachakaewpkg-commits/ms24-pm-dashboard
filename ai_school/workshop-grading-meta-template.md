## วิธีใช้งาน (3 ขั้นตอน)

**ขั้นที่ 1 — กรอกข้อมูล Workshop ในส่วน "ตั้งค่า" ด้านล่าง**  
**ขั้นที่ 2 — คัดลอก "Prompt ตรวจ" ไป paste ใน Claude Code**  
**ขั้นที่ 3 — ครั้งถัดไปใช้ "Prompt ตรวจซ้ำ" เพื่อ incremental check**

```
## ขั้นตอน

### Step 1 — สร้างหรืออ่านสถานะปัจจุบัน
ถ้ายังไม่มีไฟล์ผล → ข้ามไป Step 2 ได้เลย
ถ้ามีไฟล์ ผลการตรวจ-[SHORT_NAME].md อยู่แล้ว → อ่านเพื่อดู:
- นักเรียนที่มีสถานะ ⚠️ ในแต่ละ Level
- รายการ "ต้องแก้" ของแต่ละคน
- วันที่ตรวจล่าสุด

### Step 2 — สแกน folder หาไฟล์งานทั้งหมด
สแกนทุกไฟล์ .md ใน [FOLDER_PATH]/ ยกเว้น: [EXCLUDE_FILES]
- ไฟล์ที่ยังไม่มีในตาราง = นักเรียนใหม่ → ตรวจทุก Level ตั้งแต่ต้น
- ไฟล์ที่มีอยู่แล้วและมีสถานะ ⚠️ → ตรวจเฉพาะ Level ที่ยังค้าง

### Step 3 — ตรวจงานแต่ละ Level
สำหรับนักเรียนแต่ละคน ตรวจตามเกณฑ์ด้านล่าง:
1. ตรวจว่าครบตามเกณฑ์ Level นั้น ๆ หรือไม่
2. ✅ ผ่าน = ครบทุก requirement หลัก
3. ⚠️ ผ่านบางส่วน = มีบางข้อขาด/ทำไม่ครบ + ระบุ "ต้องแก้" ชัดเจน
4. ❌ ไม่ส่ง/ไม่ผ่าน = ไม่มีเนื้อหา Level นั้น หรือผิดพลาดหลักทั้งหมด

### Step 4 — คงสถานะ ✅ ที่ผ่านแล้ว
Level ที่มีสถานะ ✅ ไม่ต้องตรวจซ้ำ คงสถานะไว้ในรายงาน

- ตารางเกณฑ์สรุปแต่ละ Level

  สัญลักษณ์: ✅ ผ่าน | ⚠️ ผ่านบางส่วน | ❌ ไม่ส่ง/ไม่ผ่าน
- รายงานละเอียดทุกคน:
  - Level ✅: สรุปสั้น (1-3 bullet)
  - Level ⚠️: checklist + ข้อดีเด่น + "ต้องแก้" ชัดเจน
  - Level ❌: ระบุสิ่งที่ขาด
- ข้อสังเกตและข้อเสนอแนะสำหรับผู้สอน
- Prompt สำหรับตรวจซ้ำ (ฝังไว้ในไฟล์ผลด้วย)
- ภาษาไทยทั้งหมด รูปแบบ Markdown
```

## ส่วนที่ 3 — Prompt ตรวจซ้ำ (Incremental Re-check)

> ใช้เมื่อ: **นักเรียนส่งงานใหม่** หรือ **แก้ไขงานที่มี ⚠️ แล้ว**  
> วาง prompt นี้ลงในไฟล์ผล (ท้ายสุด) เพื่อใช้ซ้ำได้ทุกรอบ

---

```
ตรวจ [WORKSHOP_NAME] จากไฟล์ใน folder [FOLDER_PATH]/ โดยอิงเกณฑ์จาก [CRITERIA_FILE]

## ขั้นตอน

### Step 1 — อ่านสถานะปัจจุบัน
อ่านไฟล์ ผลการตรวจ-[SHORT_NAME].md เพื่อดู:
- นักเรียนที่มีสถานะ ⚠️ (ผ่านบางส่วน) ในแต่ละ Level
- รายการ "ต้องแก้" ที่ระบุไว้ใต้แต่ละ Level ⚠️ ของแต่ละคน
- วันที่ตรวจล่าสุด

### Step 2 — สแกน folder หาไฟล์ใหม่
สแกนทุกไฟล์ .md ใน [FOLDER_PATH]/ และเปรียบเทียบกับรายชื่อในตารางสรุป
- ไฟล์ที่ยังไม่มีในตาราง = นักเรียนใหม่ → ตรวจทุก Level ตั้งแต่ต้น
- ไฟล์ที่ timestamp ใหม่กว่ารอบตรวจล่าสุด = อาจมีการแก้ไข → เช็คซ้ำ Level ที่เป็น ⚠️
- ยกเว้น: [EXCLUDE_FILES]

### Step 3 — ตรวจซ้ำเฉพาะรายการที่ยังไม่ผ่าน
สำหรับนักเรียนที่มี ⚠️ ให้อ่านไฟล์และตรวจเฉพาะ Level ที่มีสถานะ ⚠️ โดย:
1. ดูรายการ "ต้องแก้" จาก ผลการตรวจ-[SHORT_NAME].md
2. ตรวจว่าไฟล์ที่อัปเดตได้แก้ไขรายการนั้นแล้วหรือยัง
3. ถ้าแก้แล้ว → เปลี่ยนสถานะจาก ⚠️ เป็น ✅ พร้อมบันทึกสิ่งที่ดีขึ้น
4. ถ้ายังไม่แก้ → คงสถานะ ⚠️ และระบุรายการที่ยังขาดอยู่

### Step 4 — คงสถานะ ✅ ที่ผ่านแล้ว
Level ที่มีสถานะ ✅ อยู่แล้ว ไม่ต้องตรวจซ้ำ ให้คงสถานะไว้ในรายงานใหม่

## Output ที่ต้องการ

สร้างไฟล์ ผลการตรวจ-[SHORT_NAME].md ใหม่ (overwrite) โดยมี:
- วันที่ตรวจ (วันนี้)
- เกณฑ์การตรวจสรุปแต่ละ Level
- ตารางสรุปรายบุคคล (รหัส / ชื่อ / L1–L[N] / ระดับสูงสุด)
- รายงานละเอียดทุกคน: checklist + ข้อดี + "ต้องแก้" + สรุป

  - สำหรับ Level ✅: สรุปสั้นโดยไม่ต้องรายละเอียดซ้ำ
- ข้อสังเกตสำหรับผู้สอน
- Prompt สำหรับตรวจซ้ำ (Prompt นี้เอง)
- ภาษาไทยทั้งหมด รูปแบบ Markdown
```

## ส่วนที่ 5 — ตัวอย่างการกรอก (Workshop: Python Fundamentals)

ตัวอย่าง Workshop อื่นที่ใช้ระบบนี้:

```
ชื่อ Workshop      : Python Fundamentals
Folder ไฟล์งาน    : workshop-python/
ไฟล์เกณฑ์         : python-criteria.md
ไฟล์ผลการตรวจ     : ผลการตรวจ-python.md
ไฟล์ที่ยกเว้น      : ผลการตรวจ-python.md, starter-code.md
จำนวน Level       : 3

Level ชื่อ + เกณฑ์:
  L1 — Basic Syntax:
       - Variables, Data Types, Control Flow ครบ
       - Code รันได้จริง ไม่มี Syntax Error
       - มี comment อธิบายทุก block หลัก

  L2 — Functions & OOP:
       - มีอย่างน้อย 2 Functions (parameters + return value)
       - มีอย่างน้อย 1 Class พร้อม __init__ + method
       - มี docstring อธิบาย function/class

  L3 — Data Structures & File I/O:
       - ใช้ List, Dict, Set อย่างน้อยอย่างละ 1
       - อ่าน/เขียนไฟล์ได้ (open/read/write)
       - มี error handling (try/except)
```

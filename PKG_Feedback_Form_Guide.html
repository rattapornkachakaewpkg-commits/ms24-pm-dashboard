📝 คู่มือสร้างฟอร์ม Feedback ด้วย AI

สำหรับผู้ที่ไม่มีความรู้ด้าน Dev — เอาไปประยุกต์ใช้กับงานของคุณได้ทันที

💡 กุญแจสู่ความสำเร็จ: Requirement ชัด + Prompt ละเอียด + ทดสอบทันที

1 กำหนดความต้องการให้ชัด

ก่อนจะคุยกับ AI ต้องรู้ให้ชัดเจน 5 สิ่งนี้:

ลำดับสิ่งที่ต้องกำหนดตัวอย่างจากงานนี้
1วัตถุประสงค์รวบรวม Feedback จากกลุ่มลูกค้าของทีมบัญชีและการเงิน
2กลุ่มเป้าหมายทีมรับใช้ PKG, ผู้รับใช้ทีมสาขา, สมาชิกทีมการเงินหน้างาน
3ข้อมูลที่ต้องเก็บรหัสสมาชิก, ชื่อ, ทีม, BU, คะแนน 0-5, ข้อเสนอแนะ
4ช่องทางแจ้งเตือนTelegram (ส่งทันทีเมื่อมีคนส่งฟอร์ม)
5แหล่งข้อมูลอ้างอิงGoogle Sheet Members (รหัส, ชื่อ, ทีม, BU)

2 เขียน Prompt ที่ได้ผล

❌ Prompt ที่ไม่ดี:

ช่วยทำฟอร์มสำรวจหน่อย

→ AI ไม่รู้ว่าต้องการอะไร ผลลัพธ์ไม่ตรงใจ

✅ Prompt ที่ดี (แบบที่ใช้ในงานนี้):

สร้างแบบฟอร์มรวบรวม Feedback จากกลุ่มลูกค้า
ของทีมบัญชีและทีมการเงิน PKG

รายละเอียด:
1. ฟอร์มออนไลน์บน Google Apps Script Web App
2. หน้า 1: เลือกสมาชิกจากรายชื่อ (autocomplete)
3. หน้า 2: ให้คะแนนทีมบัญชี 6 ข้อ (0-5)
4. หน้า 3: ให้คะแนนทีมการเงิน 6 ข้อ (0-5)
5. ข้อมูลสมาชิกดึงจาก Sheet "Members"
6. ข้อมูลที่ส่งบันทึกลง Sheet "Form_Responses"
7. ส่งแจ้งเตือน Telegram เมื่อมีคนส่ง
8. Theme: Light (พื้นขาว ตัวหนังสือสีเข้ม)

วัตถุประสงค์:
1. เพื่อนำ Feedback ไปพัฒนาเครื่องมือการทำงาน
2. เพื่อออกแบบระบบ "3 ผ่าน" รูปแบบใหม่
3. เพื่อรวบรวม BP/LL จากผลการดำเนินงาน

กลุ่มผู้ให้ Feedback:
1. ทีมรับใช้ PKG (ADM)
2. ผู้รับใช้ทีมสาขาทุกสาขาในทุกประเทศ
3. สมาชิกทีมการเงินหน้างานทุก BU

Google Sheet ID: [ใส่ Sheet ID จริง]
Telegram Bot Token: [ใส่ Token จริง]
Telegram Chat ID: [ใส่ Chat ID จริง]

หลักการสำคัญ: ระบุทุกอย่างที่ AI ต้องรู้ ไม่ต้องเดา

🔑 หลักการเขียน Prompt ที่ได้ผล

หลักการคำอธิบายตัวอย่าง
ระบุ Output ชัดเจนบอกว่าต้องการอะไรเป็นรูปธรรม"ฟอร์มออนไลน์ 3 หน้า" ไม่ใช่ "ฟอร์ม"
ระบุแหล่งข้อมูลบอกว่าข้อมูลมาจากไหน"ดึงจาก Sheet Members คอลัมน์ รหัส, ชื่อ, ทีม, BU"
ระบุ Scaleบอกระดับคะแนนชัดเจน"0-5 (0=น้อยสุด, 5=มากที่สุด)"
ระบุช่องทางแจ้งเตือนบอกว่าแจ้งเตือนไปที่ไหน"Telegram ส่งถึง Chat ID xxx"
ระบุ Themeบอกสไตล์ที่ต้องการ"Light Theme พื้นขาว"
ใส่ค่าจริงใส่ Sheet ID, Token, Chat ID จริงไม่ใช่ "[ใส่ชื่อ Sheet]"

3 ทดสอบและแจ้ง Bug (Interactive Loop)

หลังจาก AI สร้างฟอร์มแล้ว ให้ ทดสอบทันที แล้วแจ้ง Bug ให้ชัด:

สิ่งที่เจอแจ้ง AI ว่า
กดปุ่มไม่ได้"กดถัดไปไม่ได้ ช่องกรอกข้อมูลไม่ขึ้น"
ข้อมูลไม่แสดง"กรอก ID แล้วชื่อไม่ขึ้น"
สเกลผิด"เปลี่ยนคะแนนเป็น 0-5 (0=น้อยสุด 5=มากที่สุด)"
สีไม่ชอบ"เปลี่ยนเป็น Light Theme พื้นขาว"
แจ้งเตือนไม่มา"ส่งแล้ว Telegram ไม่แจ้งเตือน"
หัวข้อผิด"เปลี่ยนชื่อเป็น แบบฟอร์มรวบรวม Feedback"
ข้อมูลไปผิด Sheet"เปลี่ยน Spreadsheet ID เป็น [ใส่ ID ใหม่]"

💡 ทิปส์: ยิ่งแจ้ง Bug ชัดเจน เท่าไหร่ AI แก้ไขได้เร็วเท่านั้น

4 Deploy ขึ้นใช้งาน

  1. เปิด Google Apps Script (script.google.com)
  2. Ctrl+A → ลบทั้งหมด → วางโค้ดใหม่ → Ctrl+S
  3. เมนู Deploy → New deployment
  4. เลือก Web app → Execute as: Me → Who has access: Anyone
  5. กด Deploy → อนุญาตสิทธิ์ → คัดลอก URL

5 ประยุกต์ใช้กับงานอื่น

เปลี่ยนแค่ Prompt ให้ตรงกับงานใหม่:

งานตัวอย่าง Prompt
สำรวจความพึงพอใจพนักงาน"สร้างฟอร์มสำรวจความพึงพอใจพนักงาน 5 มิติ สเกล 1-5 ดึงรายชื่อจาก Sheet HR"
ประเมินหลักสูตรอบรม"สร้างฟอร์มประเมินหลักสูตร NPS 0-10 แยกตามหลักสูตร ดึงข้อมูลจาก Sheet Courses"
ลงทะเบียนกิจกรรม"สร้างฟอร์มลงทะเบียนกิจกรรม ดึงรายชื่อพนักงานจาก Sheet Members เลือกกิจกรรมจาก Sheet Events"
ขออนุมัติ/ลาพักร้อน"สร้างฟอร์มขอลา ดึงวันลาคงเหลือจาก Sheet Leave แจ้งเตือนผู้อนุมัติผ่าน Telegram"
ประเมินผล KPI"สร้างฟอร์มประเมินผล KPI รายบุคคล สเกล 1-5 ดึงเป้าหมายจาก Sheet OKRs"

🏆 สิ่งที่ได้เรียนรู้จากงานนี้ (On-the-Job Training)

เทคนิคที่ใช้

  • Full-Stack บน Google Apps Script — ทั้ง Frontend + Backend ในที่เดียว
  • HtmlService Sandbox — ต้องใช้ addEventListener แทน onclick
  • ฝังข้อมูลใน HTML — แทนการโหลดจากภายนอก (เพราะ CORS บล็อก)
  • google.script.run — เชื่อม Frontend ↔ Backend ไม่มีปัญหา CORS
  • Data Mapping อัตโนมัติ — รองรับทั้งชื่อเก่าและใหม่ของคอลัมน์
  • Telegram Notification — ส่งแจ้งเตือนทันทีพร้อมคะแนนเฉลี่ย

💡 บทเรียนสำคัญ

  • ไม่ต้องรู้โค้ดเพื่อ สั่งงาน AI — แค่รู้ว่าต้องการอะไร
  • ทดสอบทันที → แจ้ง Bug ชัดเจน → AI แก้ไขเร็ว
  • Requirement ที่ชัดเจน = Prompt ที่ดี = ผลลัพธ์ที่ตรงใจ
  • ผู้ที่เข้าใจธุรกิจ + มี AI = เหนือกว่า Dev Junior ที่เขียนโค้ดเก่งแต่ไม่เข้าใจธุรกิจ

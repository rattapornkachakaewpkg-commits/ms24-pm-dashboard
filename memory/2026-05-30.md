## CEO Dashboard - Autocomplete Bug Fix (ongoing)

### Problem History
- **Original issue**: Autocomplete ไม่ทำงานใน CEO Contract Form
- **Root cause 1**: GAS template `<?!= membersData ?>` escapes quotes inside `<script>` tags
- **Root cause 2**: `window.location.href + ?action=memberData` doesn't work inside GAS iframe (URL points to googleusercontent.com)
- **Root cause 3**: `initAC(); initExtraAC();` called at bottom of script BEFORE `google.script.run` callback returns → autocomplete initialized with empty MEMBERS
- **Root cause 4**: Multiple .gs files in project (old "Ceo code.gs") causing `buildMembersJS` error: `TypeError: Cannot read properties of undefined (reading 'length')`

### Current Fix Approach
- Changed MEMBERS loading from template injection to `google.script.run.getMembersForClient()`
- Removed `initAC(); initExtraAC();` from bottom of script (only called inside success handler)
- Removed `buildMembersJS()` function (not needed anymore)
- Added `getMembersForClient()` function in .gs
